■ プロジェクト概要

Windowsタスクスケジューラの設定内容を取得し、
24時間タイムライン形式で可視化するローカルツールを作成する。

実行環境：Python 3.7（標準モジュールのみ）

出力：ローカルHTML（外部CDN禁止）

目的：

タスクの実行タイミングを直感的に理解できるようにする

実行時間（平均）を横幅として表現する

■ 全体処理フロー
STEP 1：タスク情報の取得

Windows標準コマンドを使用

使用コマンド
schtasks /query /v /fo csv
Python実行
subprocess.check_output(...)
取得項目（最低限）

TaskName

Next Run Time

Schedule Type

Start Time

Start Date

Repeat: Every

Repeat: Until

Last Run Time

Last Result

※CSVとして取得し csv モジュールでパース

STEP 2：内部データ構造へ変換
データモデル（厳密定義）
class Task:
    name: str
    triggers: List[Trigger]
    avg_duration_sec: int  # 推定値

class Trigger:
    type: str  # ONCE / DAILY / WEEKLY / MONTHLY / REPEAT
    start_time: datetime.time
    interval_min: Optional[int]  # 繰り返し間隔
    duration_min: Optional[int]  # 繰り返し継続時間
STEP 3：繰り返し展開（重要）
目的

「1日（0:00〜24:00）」における実行イベントへ展開

出力形式
class TaskInstance:
    task_name: str
    start_minute: int  # 0〜1440
    duration_minute: int
ロジック
① 単発 / 毎日

→ そのまま1件生成

② 繰り返し（例：10分毎、1時間継続）
start = 開始時刻
end = start + duration

while t < end:
    instances.append(t)
    t += interval
STEP 4：実行時間（duration）の推定
制約

標準モジュールのみ → 履歴API不可

方法（優先順）
方法A（推奨）
schtasks /query /v

の「Last Run Time」と「Next Run Time」差分から推定

※ただし精度は低い

方法B（簡易）

固定値 fallback

DEFAULT_DURATION = 60  # 秒
方法C（拡張案）

イベントログ解析（可能なら）

wevtutil qe Microsoft-Windows-TaskScheduler/Operational

→ ただし実装難易度高

STEP 5：HTML生成
表示方式（決定仕様）
タイムライン形式

横軸：0:00〜24:00（1440px固定推奨）

縦軸：タスク一覧

HTML構造
<div class="timeline">
  <div class="row">
    <div class="label">TaskName</div>
    <div class="bar" style="left: Xpx; width: Ypx;"></div>
  </div>
</div>
CSS（ローカル埋め込み）
.timeline {
  position: relative;
}

.row {
  position: relative;
  height: 30px;
}

.bar {
  position: absolute;
  height: 20px;
  background-color: #4CAF50;
  border-radius: 4px;
}
座標変換
px_per_min = 1  # 1440px = 24h

left = start_minute * px_per_min
width = duration_minute * px_per_min
STEP 6：出力
with open("output.html", "w", encoding="utf-8") as f:
    f.write(html)
